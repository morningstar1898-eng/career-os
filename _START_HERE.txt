WHAT THIS PROJECT IS
--------------------
Career OS is a personal, automated AI job-search system: a CrewAI multi-agent
crew (Python/FastAPI backend) that runs every weekday via GitHub Actions cron,
finds ~10 target jobs/day, drafts tailored resume bullets + cover letters for
the top 5, logs everything to a Google Sheet tracker, emails a daily briefing,
and teaches a rotating daily lesson. A Next.js dashboard (Vercel) shows
briefings, an application-pipeline Kanban, interview prep, and analytics.
Backend runs as a Docker container on Azure App Service (B1, always-on),
image built by GitHub Actions and pushed to a private GHCR registry.

Agents (backend/agents + crew config): Skills Scout (finds + logs jobs as Found),
Materials Drafter (drafts materials + ATS keyword scoring), Tutor (daily lesson,
Sonnet), Interview Coach, LinkedIn Optimizer (Mondays, Sonnet), Follow-up
Tracker. All others run on Claude Haiku 4.5.

IMPORTANT CONTEXT
-----------------
- This system DRAFTS application materials and logs jobs. It NEVER
  auto-submits applications — the owner applies herself. Keep it that way.
- The owner is a current Optum/UHG employee job-hunting confidentially:
  internal Optum transfers are the top scan priority, and the repo, Docker
  image, and dashboard must all stay PRIVATE. The dashboard is password-gated
  because it reveals which companies she's applying to.
- This zip contains her real resume text, LinkedIn profile text, and target
  companies — she is sharing it knowingly for this audit. Do not suggest
  publishing any of it.
- Agent reliability is a known constraint: at higher job counts the Haiku
  apply-agent starts refusing ("can't verify these jobs"). The fix that works:
  the agent that FINDS the jobs also LOGS them (firsthand data = no refusals),
  and counts stay modest (10/day, 5 full applications). Don't recommend
  raising counts or moving logging back to the apply agent.
